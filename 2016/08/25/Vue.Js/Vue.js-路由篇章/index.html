路由配置

编程式导航：

除使用定义导航链接外，router同样提供了对应的实例方法

router.push()

  • 该方法会向histroy栈添加一个记录

  • 字符串形式：router.push(“组件名”)

    router.push("home")

  • 对象形式：router.push({ path:”路由”,query:{键：值}})

    router.push({ path:"/home",query:{name:"melody" }})

对应/home?name=melody

  • 命名形式：router.push({ name:”路由命名”,params:{变量名，变量值}})

    router.push({ name:"home",params:{name:"melody" }})

router.replace()

该方法会替换histroy的当前记录

语法与push相似，同样支持字符串-对象-命名形式

router.go()

该方法支持整数传参，意为在history记录中前进或回退多少步

router.go(-1) ---- 回退至路由记录中的前一次

路由传参

参数匹配：通过路径中的：键方式匹配路由参数

参数获取：

  • $route.params.键 —- 对应参数值

  • $route.params.page —- 5

示例：

  • 匹配的时候按顺序

    • /home/melody —– component:one

    • /home/melody/man —– component:three

    • /home/melody/sex/man —– component:four

$route对象方法

保存路由信息的对象,可在组件内部调用

  • $route.path -Str

    对应当前路由的绝对路径

"/home"

  • $route.params -Obj

    对应路由参数对象

{ 参数名：参数值，参数名：参数值 }

  • $route.query -Obj

    对应URL查询参数对象

{ name:"melody" }

  • $route.hash -Str

    对应当前路由的hash值（#）

  • $route.name -Str

    对应当前路由的命名

"home"

  • $route.fullPath -Str

    对应当前路由的完整路径

"home?name=melody"

  • $route.matched -Arr

    包含当前路由的路由记录，及路由配置对象

[{ path:"/home",component:home.. }]

路由钩子函数

全局钩子函数

  • beforeEach：路由跳转前触发

    router.beforeEach( function(to,from,next ){})

    to:即将要进入的目标路由对象

    from：即将离开的路由对象

    next()：必填项，用于导向管道中的下一个钩子节点

    next(false):中断当前导航，回退至from对应的路由

    next("指定路由")：中断当前导航，跳转至指定路由

  • afterEach：路由跳转完成时触发

    router.afterEach( function(to,from){} )

    回调中没有next参数，不需要导向次级钩子

组件内部钩子函数

  • beforeRouteEnter:路由跳转前触发

    beforeRouteEnter(to,from,next){ }

    无法访问组件实例this，钩子执行前组件实例未被创建

    next();

  • beforeRouteUpdate:路由改变 组件被复用时触发

    beforeRouteUpdate(to,from,next){ }

    如/home/2 -- /home/4跳转，能访问组件实例this

    next();

  • beforeRouteLeave:路由离开时触发

    beforeRouteLeave(to,from,next){ }

    能访问组件实例this

    next();
