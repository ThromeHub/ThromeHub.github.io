Express框架

定位：基于node平台的web应用框架

功能简介：

  • 1.提供更为简洁的API，框架中包含很多细节功能

  • 2.使开发人员更专注开发业务的代码

安装：

  • npm install express -S

步骤：

  • 1.引入express对象

  • 2.构建服务器对象

  • 3.处理响应

  • 4.监听端口并开启服务器

请求对象req：

  • 可读对象，表示浏览器请求携带的必要信息

响应对象res：

  • 可写对象，服务端响应客户端的回应信息

中间件

定位：请求与相应间封装的功能区域

语法：app.use(path,功能)

类别： 五类中间件

  • 应用中间件：app.use(功能)

  • 路由中间件：app.use(路由对象)

  • 内置中间件：app.use(虚拟目录，内置中间件对象)

  • 第三方中间件:自定义配置及调用

  • 错误中间件：app.use(function(err,req,res,next){})

应用级中间件

概括：能处理指定url的请求

原理：按照url划分独立功能块处理请求

处理完成后可通过next()方法导向次级中间件，或通过res.end()结束响应

执行顺序：中间件的执行顺序与代码编写顺序一致

异常情况：

  • 404：持续导向次级节点而不结束响应

  • 等待响应：没有结束响应

res对象扩展属性

res.download：下载资源

res.download("地址")

res.json：json自动转换字符串

res.json({ name:"Rose",age：25 })

res.redirect: 重定向 要求浏览器再发送指定请求

res.redirect("url")

res.send:响应字符串内容

res.send(`<ul>

        <li>one</li>

        <li>two</li>

    </ul>`

res.sendFile: 响应读取到的文件内容

res.sendFile( path.resolve("./main.html") );

res.sendStatus: 响应状态码

res.sendStatus(200)

  • 选用

    • 异步请求多用json函数 简化类型转换步骤

    • 同步请求使用render配合模板引擎响应页面信息

    • downloag下载 redirect重定向较为常用 避免使用send系列

express-session

作用： 响应浏览器并要求其创建指定cookie。
设置： req.session.key = value;
获取： req.session.key — value

Cookie-Session对比说明

  • 服务器响应并要求客户端创建cookie

    • 对应响应头：

      • Set-Cookie: name=melody; Max-Age=604800; Path=/main; Expires=Tue, 12 Sep 2015 06:47:48 GMT

      • Expires：设置生命周期，单位为data格式,ie兼容性较强

      • Max-Age：设置生命周期，单位为毫秒

      • Path：设置携带cookie条件，包含对应路径及子路径

    • Cookie存储位置：客户端，

      • 存储数量：随浏览器20-50不等，总大小约4kb

      • 安全性：相对不安全

    • Session存储位置：服务器内存，重启消失

      • 存储数量：没有限制

      • 安全性：相对安全，需借助cookie实现对应用户
