HTTP协议

Http作用:传输统一规范

  • 服务端与客户端间的数据格式需要统一的规定，用于解析数据使用

    采取问答模式约定数据应该以怎样的格式交互

Http特点：简单灵活 快速 无连接 无状态

  • 无连接：

    • http1.0时服务器不给客户端保持连接

    • 一次请求即是一次连接，完毕后立即关闭

    • http1.1时服务器会给客户端保持连接

  • 无状态：

    • 服务器无法获取访问者确切详细信息

  • 版本区分

    • http1.0：无连接: 服务端响应后断开连接

      对应请求头字段: connect：close

    • http:1.1:长链接：客户端关闭页面断开连接

      对应请求头字段：connect:keep-alive

  • 三次握手：

    • 客户端：请求码SYN请求建立连接

      确定码ACK：服务端

客户端：确认码ACK

- 四次挥手：

    客户端：完成码FIN

        确定码ACK：服务端

        完成码FIN：服务端

    客户端：确认ACK码

socket

Socket的设计理念是是将网络传输类比成文件的读取与写入
传送的动作被视为是写入/接收的动作被视为是读取
如此传送与接收就简化为编程人员比较容易懂的读取与写入，降低了网络编程的学习困难度。

HTTP属性

http.STATUS_CODES: 状态码列表

  • 1：正在进行中

  • 2：成功

  • 3：重定向

  • 4：客户端异常

  • 5：服务器异常

http.METHODS: restfull风格规范谓词

  • GET： 偏向于获取数据时使用

  • POST：偏向于新添数据时使用

  • PUT： 偏向于更改数据时使用

  • DELETE：偏向于删除数据时使用

  • OPTION：预检请求，跨域时携带自定义头

    浏览器在你请求之前会自动发起OPTION请求

根据服务器是否准许你跨域的答复确定你实际的跨域请求是否发出

CORS原理重申

  • COES：cross origin resource share跨域资源共享)

    • 兼容性：IE9 +

  • 原理：服务器端设置响应头，准许特定类型的跨域

    Access-Control-Allow-Origin     准许域名

    res.setHeader("Access-Control-Allow-Orgin","localhost:2000")

 Access-Control-Allow-Methods     准许请求方式

    res.setHeader("Access-Control-Allow-Methods","PUT,DELETE,GET,POST")

Access-Control-Allow-Headers     准许的自定义请求头

    res.setHeader("Access-Control-Allow-Headers","自定义请求头")

*意为所有类型
