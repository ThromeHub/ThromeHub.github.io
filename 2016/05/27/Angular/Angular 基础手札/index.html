Angularjs铺垫

  • 由谷歌团队负责开发维护的前端高级JS框架

  • 帮助我们快速构建当页面应用

  • Angular自身有很多颠覆性的特性 改变了前端的编码方式 简化了我们的操作

框架与库的区别

  • 范围不同:库针对特定方面,框架涵盖的范围更广

  • 库只是调用封装好的方法,框架会提供代码书写规则

核心思想

  • 其核心是通过指令扩展了HTML，通过表达式绑定数据到HTML。

  • Angular最大程度减少了页面上的DOM操作，让JS能专注于业务逻辑代码

    通过简单指令结合页面结构与逻辑数据，实现组件化，降低维护成本

常用指令

  • ng-app 模块挂载

    • 告诉Angularjsjs当前页面中需要控制的范围

    • 告诉angularjs当前页面由哪个模块来管理

    • 页面加载完成后Angularjs会自动在页面中查找该指令，如未查询到Angularjs将不会启动

  • ng-model 双向数据绑定

    • ng-model = “tode” 可绑定变量

  • ng-init 初始化变量

    • ng-init = “tode=’melody’” 很少使用
  • ng-click 绑定点击事件

作用域

  • 局部作用域

    • angularjs中控制器控制的区域就是一块局部作用域

    • 本质上$scope代表着局部作用域

    • 相邻控制器之间是独立的 互不影响的

      • 作用域嵌套的请求下 数据是逐层向外部查找的 直到全局作用域

  • 全局作用域

    • angularjs中$rootScope代表的就是全局作用域

    • 挂载到全局作用域中的数据 每个控制器中都可以拿到

    • 可以注入到run方法中使用 用作项目初始化

      • run方法优先于控制器方法执行且仅执行一次

MVC模型

  • module模型：与数据库交互通信

  • views视图：以友好的方式展示数据

  • controller控制器：组织调度相应的数据模型

  • 以登录为例

    • 模型：接收控制器传来的用户信息并进行校验的业务逻辑

      • 返回true/false

    • 控制器：接收用户在界面填写的信息

      • 将用户信息发送给模型

    • 视图：给用户提供表单，接收用户输入信息

      • 将信息提交给控制器，根据控制器返回的 信息响应用户页面

路由传递参数

  • 在实际的项目中,访问文章详情页面的时候,一般地址栏上都会显示当前文章的ID

  • 让控制器知道我们当前访问的是哪篇文章,得到文章ID,去请求具体的文章数据

  • 代码示例

    angular.module("main",["ngRoute"])
  .config(function($routeProvider){
    $routeProvider.when("/锚点/:id",{
      template:"<div> "用户的ID为：" + id </div>",
      controller:"con"
    })
  })
  .controller("con",function($scope,$routeParams){  
    $scope.id = $routeParams.id；      //$scope.id为锚点末尾的参数
  })
