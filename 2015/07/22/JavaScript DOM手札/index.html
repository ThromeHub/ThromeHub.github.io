JavaScript DOM手札

    document        文档

    object          对象

    model           模型

    DOM给我们提供一些方式，操作页面内容
节点 
  当页面加载中，首先根据当前页面的html结构生成结构树

  js会根据结构树生成同样的节点树（每个标签会以对象形式呈现）

      这个树称为节点树，作用是方便用户操作

  节点树准许我们根据每个节点的上下级/同级间的关系进行访问（给我们提供访问方式）



  节点的分类：

      1.元素节点：标签（最常用）

      2.属性节点：标签的属性

      3.文本节点：标签的内容

      4.根节点：document（document实际上是DOM中的顶级对象）


  节点获取.

      访问父级元素

      子节点.parentNode


      访问子节点：    返回数组

      子节点.children

      问题：在ie低版本中可以获取注释节点




      追加子节点   剪切过去

      内部最后：

          父节点.appendChild(添加节点)

      指定位置：

          父元素.insertBefore(添加节点，位置节点)

          父元素.insertBefore(添加节点，null)

          如果位置节点为空，默认插入最后位置
获取操作 
  id获取：     注意大小写

        document.getElementById ( "id名" )

        如果获取结果为null，表示获取失败

        可能情况

            - 大小写问题

            - id 拼写及存在问题

            - 执行顺序问题（script在head中）


        执行顺序问题解决

            window.onload = function(){ 

                document.getElementById ( "id名" )

            }

            内部所有代码在页面加载后执行




  标签名获取：      注意Element‘s’

        document.getElementsByTagName ( "标签名" )

        返回的是一个伪数组（不能使用数组方法）

        前边的document对应window，可以换成别的标签


        操作特定元素：

            var tag = document.getElementsByTagName ( "标签名" );

            tag[0].innerHTML = 内容；


        操作每个元素：

            var tag = document.getElementsByTagName ( "标签名" );

            for(var i = 0; i < tag.length; ++i){

                tag[i].innerHTML = i;

            }




  类名获取：     与标签名相似  

        HTMLObject.getElementsByClassName("类名");


        ie9以下不支持该方法，也没有替代，需要自己书写功能实现
        

  全局获取：

        document.querySelectorAll(选择器)

        取得的是静态数组，不会实时更新


  body的获取：

        document.body
属性操作 
  取值：htmlObject.属性名

  赋值：htmlObject.属性名 = 属性值

  自定义属性：htmlObject.定义属性 = 属性值
css样式操作： 
   
      HTMLObject.style.样式

      使用js生效的样式会以行内嵌入的方式完成


  获取：如果不是行内样式，获取不到结果

        可以使用getComputStyle方法获取：

            getComputedStyle（box，null）.width

            参数A:要获取的元素

            参数B：伪元素，如果有书写字符串的为元素，如果没有可以设置null

            返回值为对象



        ie低版本不支持

        可以使用标签.currentStyle属性功能与上式相同

            box.currentStyle.width


  赋值：
      HtmlObject.style.fontSize = "18px";

          在css样式中，名称内含有-的样式，需要修改为驼峰命名法

      HTMLObject.style.cssText = "width:100px; height:40px"

          可直接设置css属性



  简化函数
      定义函数：
          function setStyle(tag, data){

              for(var k in data){

                  tag.style[k] = data[k];
              } 
          }

      调用：
          setStyle(htmlObject,{

              width:"100px",

              height:"70px",
          });
Class操作 
  当JS需要样式较多时推荐使用

  取值：

        HtmlObject.className

  添加类：

        HTMLObject.className = 类名

  更替类：

        HtmlObject.className.replace("目标"，"替换内容")

        字符串方法不改变原值,返回值为修改后的值



        当元素具有类似命名时，用replace操作时会出现问题

        1.取元素的类，字符串格式

              var str = HtmlObject.className;

        2.字符串.split（" "）  以空格分割字符串组成数组

              var arr = str.split(' ');

        3.检索数组中的字符串‘hide’，并修改

              var index = arr.indexof("hide");

              arr[index] = "show"；

        4.数组.join(" ")  以空格组合数组成字符串

              var str = arr.join(" ");

        5.类的返还，字符串格式

              HTMLObject.className = str;


        更多练习实例参考：

            tab动态窗口.html    按钮变色练习.html
文本操作 
  innerHTML 包含标签及文本

    htmlObject.innerHtml = "标签可以被获取"

          ----  标签 可以 被获取

          设置innerHTML会覆盖原本标签内内容

    解决：

          btn.innerHTML += "新内容";

          依然是复制，只是内容更新



  innerText 包含文本 不识别标签，性能佳

        htmlObject.innerText = "标签可以被获取"

          ----  标签可以被获取

          不能转译标签，标签会以字符串方式添加

    textContent   标准属性，其他浏览器支持，ie678低版本不支持

事件 
  事件源：  触发事件的元素

  事件类型： 触发事件的方式

  事件效果： 内部为事件触发后需要做的事情



  方式A:

    事件源.事件类型 = function（）{

          事件处理程序
        }

    事件处理程序函数并不是由函数调用进行执行的，而是由用户操作触发的

    当我们给指定标签设置函数值后，事件机制会实时检测这个事件



  方式B：

    标签内添加属性方式

          li onclick = 'add( )'

    在script中添加对应函数

          function add(){ }




      onclick事件细化

        onmousedown 鼠标按下

        onmouseup 鼠标按起



    阻止默认点击效果：

        在点击事件末尾添加，如果阻止未生效说明前边有错

        retrun false  


    阻止文字选中的效果：

        添加 onmousedown onmouseup事件

        返回 retrun false   



    阻止方法：
      window.event.cancelBubble = true;

          谷歌和IE8支持，火狐不支持

      e.stopPropagation()

          谷歌和火狐支持，IE8不支持

DOM添加节点 
  document.write      页面添加文本及标签

  - 问题：页面中的所有内容被替换为新内容

        obj.onclick = funciton(){

            document.write("内容");
        }

  - 原因：页面的首次加载会开启一个输出流，当页面加载完毕后，输出流随之关闭。

        页面加载后使用document.write会开启一个新的输出流，导致旧的输出流被覆盖

  - 问题：可以在加载过程中使用，但是不能指定位置创建内容




  innerHTML     更改对象文本及标签内容

  -适用：内部结构比较复杂时 

  -问题：对原内容进行覆盖

        使用+=可以保留数据，但不能保留事件

  -问题：性能低

        向div中添加2000个空div

        打印执行时间 ----- 2+s

              console.time("标识符"0

                执行内容

              console.timeEnd("标识符")

  -解决：var str = "Object.innerHTML";

        每次循环执行

            str += "添加内容"

        循环结束后统一赋值 

            Object.innerHTML = str

        通常配合appendChild 或 insertBefore

        打印执行时间 ----- 0.002s

              console.time("标识符"0

  动态创建元素

      会导致重新发生页面的渲染效果

      重排：当元素大小改变或者位置改变

      重绘：更改背景图景色，内容文本


      重排必定引发重绘

      我们在操作过程中需要尽量减少重排次数


  优化方案

      借助documentFragment文档片段

            可以解决重排次数过多的问题

      特性：添加到页面中式，自身不会被添加，却可以 添加自身的子元素
      

  需要对页面中的某元素进行大小的修改

  可能这个元素的高度自适应，通过内部多次添加子元素，撑开父元素会引发重排

  先对这个元素进行display：none，只会在显示和隐藏时发生两次重排
