函数与Function

-> length 属性

  在定义函数的时候, 函数有length属性, 用于描述函数的参数个数

    function wode() { }             wode.length -- 0

    function wode( item ) { }        wode.length -- 1



-> arguments 对象

      该对象是一个伪数组, 用于获得所有的传入参数. 实际函数调用时传入的所有参数都会依次存储在arguments中.


利用该属性可以实现函数的多种调用结果( 函数重载 ).

        function wode() {

              console.log( arguments ); 

           }

           wode( 1, 2, 3 );                ---- [1,2,3]            

        wode( 'a', true, 1, { name: "哈哈" });        ---- ["a",true,1,{ name:"哈哈" }]


      函数重载, 原本不是js中的语法特性, 是 C++ 引入的语法特性。

通过arguments判断参数的类型及数量，在函数内部执行不同代码，实现函数重载的模拟




-> callee 与 caller

  1> callee 是 arguments 的方法, 表示当前函数

      function wode() {

        arguments.callee();         ---- wode()

      }


  2> caller 是函数的属性方法, 在函数套用过程中表示调用该函数的引用

      function wode() {

        todo();

      }

      function todo() {

        console.log( todo.caller );        ---- wode的函数体

      }

Function 的使用

作用：  可用于创建函数,将字符串作为Js代码执行


语法:   new Function( "参数", "参数", .. "参数", "函数体" )


注意:   Function 的参数全部是字符串, 最后的参数用于构成函数体, 其余参数为该函数的参数.

    用Function无法创建全局变量



实例: 求两个数字的和, 并将结果返回的函数

    var getSum = new Function( "numone", "numtwo", "return numone + numtwo");



实例: 请求返回的json格式字符串就可以使用此法直接执行 

      ( new Function ( "return " + jsondata ) ) ( );

面向对象小结与补充

-> 包装对象( Number, String, Boolean )

思考: 字符串的方法是谁提供的?

    由于 js 的基本类型不是对象, 为了让它可以方便操作, js 作者为它们设计了包装对象来提供各种方法.

    在代码的执行过程中, 如果需要使用方法, 会在代码中临时创建对应的对象, 并在调用方法后释放.




实例: "wode,tode".split(",")

      1> 创建一个 String 的实例( new String("wode,todo" ) ).

      2> 调用实例对应的 split 方法, 分隔字符串并返回一个数组.

      3> 在返回数组的之后, 释放该对象.

如果要取得对应包装对象中存储的值, 使用 valueOf 方法.

-> 面向对象特征( 三大特征 )

  1> 抽象性

    其原意是说将你研究的目标上抽取你需要的特征, 然后将这些特征组合起来表征这个具体的事务. 


  2> 封装性

    所谓的封装性就是将复杂的逻辑, 或需要隐藏的数据保护起来, 不让使用者看到, 了解到. 

    只预留简单的接口被使用.如函数, 就是封装性的体现. 


  3> 继承性

    拿来主义. 强调自己原来是没有的. 然后有了.


  4> 多态性

    原意: 一个对象调用相同的方法, 会展示出不同的效果.

    最简单的多态性就是 js 中的 toString 方法.
