Redux-React

依赖:

  • npm i react-redux –save

  • npm i redux-devtools –save-dev

三项原则：

  • 单数据源：状态树的唯一，即store存储对象的唯一性

  • 状态只读性：触发action是改变状态的唯一方法

  • 纯函数事件：即固定传参返回固定结果的定向函数，每次返回崭新的state对象

Connect —- 链接组件与Redux

connect( mapStateToProps, mapDispatchToProps, mergeProps, options )( component )

connect方法的返回值为函数，其作用是封装传参组件使其能够关联Redux状态树

connect方法的传参貌似很复杂，实际上仅前两项常用


参数mapStateToProps:负责输入，将Store中的state转化为组件props

参数mapDispatchToProps：负责输出，将事件生成器转化为组件props，使组件具备更新state的能力

参数mergeProps:负责整合/过滤组件props

目前组件的props来源包含前两种方法生成的props以及组件自身的props，分别对应方法的三个参数

中间件与异步操作

定位：重新封装dispatch方法并拓展其功能

引用：作为createStore方法的第三个参数引用


Redux同步操作：派发事件Action，立即调用监听器Redux更新Store

Redux异步操作：能配合请求套用派发事件Action，调用监听器Redux更新Store

说明：
1.dispatch方法仅支持对象参数，无法实现异步派发操作，需要promise/Thunk中间件的支持
2.由于中间件的调用采用数组操作的形式，故有次序要求

日志中间件 redux-logger 需要排列在最后

  • import Logger from “reudx-logger”

  • const store = createStore( reducer, applyMiddleware( Logger ))

效果：会打印前后状态及对应的事件

  • prev state —- { name:”melody” }

  • action —- { type:”change_name”, name:”Sakuiro” }

  • next state —- { name:”Sakuiro” }
