Vue.js

定位：以数据驱动和组件化思想构建的MVVM库。

  • 相比于Angular更加简洁与易于理解

官网: http://cn.vuejs.org/

手册： http://cn.vuejs.org/api/

Vue-Angular简要对比

Vue：

  • 指令形式为 v-

  • 属性及方法均挂载在组件上

  • 适用: 移动端项目

  • 发展：vue发展迅猛，github上start数量已超越Angular

  • 组件化：将项目划分为独立的结构组件（vue文件），具备完整的样式 视图及逻辑代码

Angular：

  • 指令形式为 ng-

  • 属性及方法均挂载在作用域对象$scope上

  • angular由google维护

  • 合适: pc端项目

  • 模块化：依据功能将项目划分为独立的结构部分，每一模块能够独立实现某特定功能

两者均不兼容低版本IE

数据观测

Angular数据观测：

  • 采用脏值检查机制，指令内置名为watcher的应用用于观测作用域内的数据变化

    当数据更新或页面刷新时，Angular会遍历当前页面中所有的watcher并逐一对其求值更新

  • 特点：

    • 1.数据庞大时应用的性能会受到影响，且很难优化

    • 2.触发数据观测的条件有限，对于Js原生方法的数据更新需要手动调用$apply方法更新视图

Vue.js数据观测：

  • 本质上调用ES5的Object.defineProperty（）方法，参数依次为对象-属性-配置对象

    配置对象内置set及get方法用于监听对应数据的获取及赋值操作，从而能独立观测数据关联视图

Vue生命周期

  • beforeCreate: 实例化之前

  • created: 实例化完成

  • beforeMount: 实例挂载，模板渲染前

  • mounted: 实现挂载，模板渲染后 *（ready V1.0）

  • beforeUpdate: 组件更新之前

  • updated: 组件更新完毕 *

  • beforeDestroy: 组件销毁前

  • destroyed: 组件销毁后

修饰符

  • @事件类型.stop = “函数”:

    • 阻止事件冒泡，等同于event.stopPropagation()

    • 事件冒泡：子元素在自身事件执行结束后，会执行父元素相同类型的事件

  • @事件类型.prevent =”函数”:

    • 阻止默认事件，等同于event.prevent

    • 默认行为：系统默认的操作，如右键出现的菜单，contextmenu为右键点击事件

  • 按键修饰符

  • 常用修饰符：

    • enter tab delete esc space

    • left right up down
